      31             :  *
      32             :  * \details Given a point with cartesian coordinates \f$\xi^i\f$, let the polar

      34             :  * defined in the usual way: \f{align}{
      35             :  * \xi^0 - x_c^0 &= r sin(\theta) cos(\phi)\\
      36             :  * \xi^1 - x_c^1 &= r sin(\theta) sin(\phi)\\
      37             :  * \xi^2 - x_c^2 &= r cos(\theta)
      38             :  * \f}
      39             :  * The shape map distorts the distance \f$r\f$ between the point and the center
      40             :  * while leaving the angles \f$\theta\f$, \f$\phi\f$ between them preserved by
      41             :  * applying a spherical harmonic expansion with time-dependent coefficients
      42             :  * \f$\lambda_{lm}(t)\f$. An additional domain-dependent transition function
      43             :  * \f$f(r, \theta, \phi)\f$ ensures that the distortion falls off correctly to
      44             :  * zero at the boundary of the domain. The shape map maps the unmapped
      45             :  * coordinates \f$\xi^i\f$ to coordinates \f$x^i\f$:
      46             :  *
      47             :  * \note The quantities stored in the FunctionOfTime are not the
      48             :  * complex spherical-harmonic coefficients \f$\lambda_{lm}(t)\f$, but
      49             :  * instead are the real-valued SPHEREPACK coefficients
      50             :  * \f$a_{lm}(t)\f$ and \f$b_{lm}(t)\f$ used by YlmSpherepack.  The
      51             :  * relationship between these two sets of coefficients is
      52             :  * \f{align}
      53             :  * a_{l0} & = \sqrt{\frac{2}{\pi}}\lambda_{l0}&\qquad l\geq 0,\\
      54             :  * a_{lm} & = (-1)^m\sqrt{\frac{2}{\pi}} \mathrm{Re}(\lambda_{lm})
      55             :  * &\qquad l\geq 1, m\geq 1, \\
      56             :  * b_{lm} & = (-1)^m\sqrt{\frac{2}{\pi}} \mathrm{Im}(\lambda_{lm})
      57             :  * &\qquad l\geq 1, m\geq 1.
      58             :  * \f}
      59             :  * The FunctionOfTime stores coefficients only for non-negative \f$m\f$;
      60             :  * this is because the function we are expanding is real, so the
      61             :  * coefficients for \f$m<0\f$ can be obtained from \f$m>0\f$ coefficients by
      62             :  * complex conjugation.  Here and below we write the equations in
      63             :  * terms of \f$\lambda_{lm}(t)\f$ instead of \f$a_{lm}(t)\f$ and
      64             :  * \f$b_{lm}(t)\f$ because the resulting expressions are much shorter.
      65             :  *
      66             :  * \f{equation}{
      67             :  * x^i = \xi^i - (\xi^i - x_c^i) f(r, \theta, \phi) \sum_{lm}
      68             :  * \lambda_{lm}(t)Y_{lm}(\theta, \phi).
      69             :  * \f}
      70             :  *
      71             :  * The inverse map is given by:
      72             :  * \f{equation}{
      73             :  * \xi^i = x_c^i + (x^i-x_c^i)*(r/\tilde{r}),
      74             :  * \f}
      75             :  * where \f$\tilde{r}\f$ is the radius of \f$\xi\f$, calculated by the
      76             :  * transition map. For more details, see
      77             :  * ShapeMapTransitionFunction::original_radius_over_radius .
      78             :  *

      80             :  * \f{equation}{
      81             :  * v^i = - (\xi^i - x_c^i) f(r, \theta, \phi) \sum_{lm}
      82             :  * \dot{\lambda}_{lm}(t)Y_{lm}(\theta, \phi).
      83             :  * \f}

      86             :  * \f{equation}{
      87             :  * \frac{\partial}{\partial \xi^j} x^i = \delta_j^i \left( 1 - f(r, \theta,
      88             :  * \phi) \sum_{lm} \lambda_{lm}(t)Y_{lm}(\theta, \phi)\right) + (\xi^i - x_c^i)
      89             :  * \left(\frac{\partial}{\partial \xi^j} f(r, \theta, \phi) \sum_{lm}
      90             :  * \lambda_{lm}(t)Y_{lm}(\theta, \phi) + f(r, \theta, \phi) \sum_{lm}
      91             :  * \lambda_{lm}(t) \frac{\partial}{\partial \xi^j} Y_{lm}(\theta, \phi) \right).
      92             :  * \f}
      93             :  *
      94             :  * The inverse Jacobian is computed by numerically inverting the Jacobian.
      95             :  *
      96             :  * For future optimization, the `interpolation_info` objects calculated in all
      97             :  * functions of this class could be cached. Since every element should evaluate
      98             :  * the same grid coordinates most time steps, this might greatly decrease

     102             :  * higher.
     103             :  */
